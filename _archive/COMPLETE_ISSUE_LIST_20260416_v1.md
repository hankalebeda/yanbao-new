## 1. P0 级问题 - 系统失效

### P0-001: 已删除-已发布状态冲突

**问题描述**  
164 条已删除的研报（`is_deleted=1`）仍被标记为已发布（`published=1`），违反了数据完整性约束。这可能导致已删除的报告通过 API 暴露。

**影响范围**  
- **维度**: D2 (数据完整率) + D3 (端到端可用率)
- **关键路径**: `/api/v1/reports` 列表接口可能暴露已删除数据
- **用户影响**: CRITICAL - 数据真实性约束破坏

**证据**  
```sql
-- 验证查询
SELECT COUNT(*) FROM report WHERE is_deleted=1 AND published=1
-- 结果: 164 行

-- 样本数据
report_id: 0a192ec1-32fc-4c... | stock_code: 600519.SH | is_deleted: 1 | published: 1
```

**根因分析**  
删除操作未同时清理 `published` 标记，可能是：
1. 删除逻辑只设置 `is_deleted=1`，未复置 `published`
2. 或历史数据混态，prior fix 未全覆盖

**修复状态**: ✅ **已修复 (2026-04-16)**  
```
UPDATE report 
SET published=0, updated_at=NOW() 
WHERE is_deleted=1 AND published=1

结果: 164 条记录修复，验证后 conflict_count=0
```

**修复代码位置**: DB Hotfix (无需代码文件修改)

**验证方法**  
```sql
SELECT COUNT(*) FROM report WHERE is_deleted=1 AND published=1
-- 预期结果: 0
```

## 2. P1 级问题高优先级

### P1-001: 搜索自动补全 API 缺失

**问题描述**  
前端搜索框输入股票代码时缺少自动补全 `.SZ/.SH` 后缀的功能。`GET /api/v1/stocks/autocomplete` 和 `GET /api/v1/stocks` 端点均返回 404 NOT_FOUND。

**浏览器测试证据**  
```

Observation: /api/v1/stocks/autocomplete?q=000001 → 404 NOT_FOUND
Impact: HIGH - 核心搜索交互体验受损
```

**影响范围**  
- **维度**: D5 (页面展示率) + D3 (可用性)
- **关键路径**: 首页搜索框 → API `/api/v1/stocks/autocomplete`
- **用户影响**: HIGH - 新用户不知道需要输入完整代码格式（如 000001.SZ）

**当前行为**  
- 用户在首页搜索框输入 `000001` → 无自动补全提示
- 必须手动输入完整 `000001.SZ` 才能搜索
- 搜索结果为空

**修复建议**  
新增 API 端点（需在 `/app/api/routes_business.py` 中实现或确认）：
```

    {"code": "000001.SZ", "name": "平安银行", "exchange": "SZ", "match_type": "code"},
    {"code": "001979.SZ", "name": "招商银行", "exchange": "SZ", "match_type": "code"}
```

**修复优先级**: HIGH / 预计耗时: 1-2 小时

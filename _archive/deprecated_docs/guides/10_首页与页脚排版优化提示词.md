# 首页与页脚排版优化提示词

> **用途**：针对 demo/index.html 的市场横幅、Hero 区、四维度绩效（hero-cred）及页脚完整声明的排版优化。  
> **问题**：市场/Hero/绩效区排版不理想；完整声明过于居中、两侧空白过大。  
> **Skills**：frontend-design、responsive-design、visual-design-foundations、interaction-design。

---

## 一、完整提示词（可直接复制使用）

```
【任务】对 demo/index.html 的市场横幅、Hero 区、四维度绩效及页脚完整声明进行排版优化，使布局更美观、留白均衡、信息层次清晰。必须调用 frontend-design、responsive-design、visual-design-foundations、interaction-design 四个 skill。

【待优化文件】
- demo/index.html
- demo/demo.css

【优化区域与要求】

一、今日市场横幅（.market-banner）
现状问题：标签与数值竖排分散，专业名词解释（量能比、MA20）与主文案挤在一起，视觉层次弱。
优化方向：
1. 每项采用「标签 + 数值」横排或卡片式布局，标签与解释（term-tip）分离，数值突出
2. 可选：将解释改为悬停 tooltip 或放在标签下方小字，避免「量能比 相对5日均量」挤在一行
3. 四项网格保持 2×2 或 4 列，但每格内排版更紧凑、层级更清晰
4. BULL 积极 徽章与今日市场可合并为一行展示，或保持独立格但优化对齐

二、Hero 区（.hero）
现状问题：kicker、标题、tagline、搜索框、常用链接、四维度绩效 堆叠过密或过疏，整体节奏不协调。
优化方向：
1. 信息层级：kicker → 主标题 → tagline 形成清晰的视觉流，间距用 Design Tokens 统一
2. 搜索框与常用链接：可考虑左右或上下分组，常用链接与搜索框视觉关联更强
3. 四维度绩效（.hero-cred）：当前「胜率 ≥55% · 盈亏比 ≥1.5 · 年化 Alpha（超额收益）≥10%」排版松散
   - 建议：改为小型指标卡片行（3 个 mini-card 并排），或使用更紧凑的 inline 展示，数字与单位不割裂
   - 年化 Alpha 与「超额收益」解释的关系要清晰，避免「年化 Alpha超额收益」连在一起
4. 「高级区展示……」与绩效指标可用分隔线或换行区分，避免混在一行过长

三、页脚完整声明（.footer-disclaimer、.footer-full）
现状问题：声明文案 max-width: 640px、居中，导致两侧大片空白，视觉上「太集中」。
优化方向：
1. 页脚内容区宽度与主内容区（.container）对齐，或略宽，避免声明独占窄条
2. 可使用 .footer-inner 或 .footer-disclaimer 设置 max-width: 与 container 一致（如 1200px），并 text-align: left 或 justify，使段落自然铺满
3. 展开后的 .footer-full 段落采用合理的行宽（45–75 字符/行可读性最佳），可通过 max-width: 65ch 或与 container 同宽实现
4. 管理后台链接与声明 summary 的排版保持协调，不突兀

四、通用约束
- 沿用 demo.css 的 Design Tokens（--space-*、--fs-*、--radius、语义色）
- 375px 无横向溢出，小屏可单列或折叠
- 专业名词解释保留，可调整展示方式（inline 小字 / tooltip / 换行）

【Skills 调用】
- frontend-design：视觉层次、留白节奏、整体美观
- responsive-design：断点、网格、移动端适配
- visual-design-foundations：排版、间距系统、可读性
- interaction-design：hover 态、展开/折叠体验

【输出】
1. 修改后的 demo/index.html 相关片段（若 HTML 结构需调整）
2. 修改后的 demo.css 中 .market-banner、.hero、.hero-cred、.site-footer、.footer-disclaimer、.footer-full 等样式
3. 简短说明：做了哪些排版改动、解决了哪些视觉问题

【验收】
- 市场横幅：标签与数值关系清晰，专业名词解释不挤占主文案
- Hero：层级分明，四维度绩效易读不松散
- 页脚：声明区与主内容宽度协调，两侧无过大空白
```

## 二、排版思路参考（非强制）

### 市场横幅
```
方案 A：标签 + 数值横排
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ 今日市场     │ 沪深300近20日 │ 量能比       │ MA20        │
│ BULL 积极   │ +2.3%       │ 1.12        │ 支撑        │
│             │             │ 相对5日均量  │ 20日均线    │
└─────────────┴─────────────┴─────────────┴─────────────┘

方案 B：解释放标签内小字
量能比 · 1.12  （相对5日均量 用更小字号、更淡色）
```

### Hero 四维度绩效
```
方案 A：mini 卡片行
┌──────────┐ ┌──────────┐ ┌──────────────────┐
│ 胜率     │ │ 盈亏比   │ │ 年化 Alpha       │
│ ≥55%    │ │ ≥1.5    │ │ ≥10% 超额收益    │
└──────────┘ └──────────┘ └──────────────────┘

方案 B：紧凑 inline + 分隔
胜率 ≥55%  ·  盈亏比 ≥1.5  ·  年化 Alpha ≥10%（超额收益）  |  高级区展示…
```

### 页脚
```
当前：.footer-disclaimer max-width: 640px; margin: 0 auto; → 窄条居中
优化：max-width: min(720px, calc(100% - 2 * var(--container-pad))); 或与 .container 同宽
     text-align: left; 使段落自然铺满
```

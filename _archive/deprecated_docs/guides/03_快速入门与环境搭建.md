# 28 快速入门与环境搭建

## 1. 适用范围
用于新机器首次搭建本项目开发与运行环境（Windows）。

## 2. 前置条件

> ⚠️ **第一次部署必读**：本系统有以下硬性前提，缺少任意一项会导致部分功能降级运行：
> 1. **通达信金融终端**（本地历史因子计算）
> 2. **Ollama + qwen3:8b 模型**（本地 LLM 兜底）
> 3. **已有可用 AI 接口**（**不需配置 AI API Key**）：ai-api 四个 Web API（手动登录）或 Codex/Gemini CLI 或 Ollama；详见 `docs/core/08_AI接入策略.md`

1. 已安装 Git、Python 3.10+（建议 3.11）、Node.js（如前端需要）。
2. 已安装 NVIDIA 驱动（如需本地 Ollama 推理）。
3. 具备仓库读取权限与必要 API 凭据。

## 3. 初始化步骤
1. 克隆项目并进入目录。
2. 创建虚拟环境并激活。
3. 安装 Python 依赖（`pip install -r requirements.txt`）。
4. 复制环境变量模板：`.env.example -> .env`，填写必填项（见步骤 3.2）。
5. **安装通达信**（重要，见步骤 3.3）。
6. 启动 Ollama：`ollama serve`。
7. 拉取模型：`ollama pull qwen3:8b`（当前生产标准，见 ADR-002；`deepseek-r1:8b` 已废弃，仅供历史测试用）。

### 3.1 推荐命令（Windows）

1. 创建与激活虚拟环境：
   - `python -m venv .venv`
   - `.\.venv\Scripts\Activate.ps1`
2. 安装依赖：
   - `pip install -r requirements.txt`
3. 配置环境变量：
   - 复制 `.env.example` 为 `.env`
   - 生产建议设置 `INTERNAL_API_KEY`（内部接口鉴权，详见 `docs/guides/02_运维手册.md` §3 与 `docs/core/05_API与数据契约.md` §0.3）

### 3.2 必填 .env 配置项

| 变量名 | 说明 | 优先级 |
|--------|------|--------|
| `TDX_INSTALL_DIR` | 通达信安装目录（如 `C:\new_tdx`）| **必填**（历史因子计算） |
| `INTERNAL_API_KEY` | 内部接口鉴权 Key | 生产推荐 |
| `WEIBO_COOKIE_SUB` | 微博 Cookie 的 SUB 字段 | 推荐（热搜质量更高） |
| `ALERT_WEBHOOK_URL` | 企业微信/钉钉告警 Webhook | 可选 |

> 获取微博 Cookie：登录 weibo.com → 浏览器 F12 → Network → 任意请求 → Request Headers → Cookie → 复制 `SUB=...` 的值

### 3.3 安装通达信（历史因子预计算必需）

本系统的"历史信号先验"（近90日胜率/ATR分位/量比分位）依赖通达信本地 `.day` 文件。

**安装步骤**：
1. 下载通达信金融终端（官网：www.tdx.com.cn）
2. 默认安装路径：`C:\new_tdx`
3. 打开软件，登录后等待数据同步完成（首次约1~2小时）
4. 确认目录存在：`C:\new_tdx\vipdoc\sh\lday\` 和 `C:\new_tdx\vipdoc\sz\lday\`
5. 在 `.env` 中设置：`TDX_INSTALL_DIR=C:\new_tdx`

**如果暂时不安装**：系统进入"冷启动模式"，历史先验使用默认值 0.52（略优于随机基准），研报仍可生成，但历史胜率分析精度下降。在 `report_data_usage.tdx` 中会标注 `missing`。

## 4. 本地启动
1. 启动后端服务。
2. 访问健康检查接口与报告页面。
3. 执行最小链路：生成 -> 查询 -> 结算。

### 4.1 启动与最小链路（建议固定端口）

1. 启动（固定端口 8010，需先激活虚拟环境）：
   - `python -m uvicorn app.main:app --host 0.0.0.0 --port 8010`
2. 健康检查：
   - `GET http://127.0.0.1:8010/health`
3. 内部健康（若设置了 `INTERNAL_API_KEY`，需带 `X-Internal-Key`）：
   - `GET http://127.0.0.1:8010/api/v1/internal/llm/health`
   - `GET http://127.0.0.1:8010/api/v1/internal/metrics/summary`
4. 研报生成与查询：
   - `POST http://127.0.0.1:8010/api/v1/reports/generate`，请求体 JSON：`{"stock_code":"600519.SH","run_mode":"daily"}`（契约见 `05_API与数据契约` §2.0）
   - `GET http://127.0.0.1:8010/api/v1/reports/{report_id}`
5. 列表与看板（站点必需能力）：
   - `GET http://127.0.0.1:8010/api/v1/reports?page=1&page_size=20`
   - `GET http://127.0.0.1:8010/api/v1/predictions/stats`

## 5. 最小验收
1. 服务进程启动成功且无阻断报错。
2. 核心 API 返回成功或可解释错误码。
3. `pytest -q tests` 可执行（当前是否全绿以 `docs/planning/07_生产就绪差距与执行计划.md` 为准）。

## 6. 代理环境与国内数据源

若本机使用 Clash/V2Ray 等系统代理：
- **北向资金、东方财富**等国内站点：代码已内置 `bypass_proxy()`（`app/core/proxy_utils.py`），请求前会自动关闭代理，避免 `ConnectionResetError`。
- **Ollama 模型拉取**：需走代理才能访问境外镜像时，可临时关闭代理 bypass 或对 `ollama` 进程单独设置代理。

## 7. ai-api 启动（可选）

ai-api 可独立于主服务运行，供研报推理可选接入或人工辅助分析：

- **路径**：项目根下 `ai-api/`，包含 `chatgpt_web/`、`deepseek_web/`、`gemini_web/`、`qwen_web/` 及 `webai/` 统一入口
- **接口文档**：`ai-api/WEB_AI_UNIFIED_API.md`
- **启动**：需先安装 Playwright、手动登录对应网站（Chrome/Edge 用户数据目录），按 `ai-api/` 各子模块 README 或 `webai` 入口说明启动
- **端口**：以实际启动为准，主服务引用时需配置对应 URL

> **本部署不需 AI API Key，已有可用接口**：ai-api 四个 Web API（需手动登录）、Codex/Gemini CLI、本地 Ollama。未启动 ai-api 时，研报生成会降级到本地 Ollama。

## 8. 常见问题
1. 模块缺失：重新安装依赖并确认激活虚拟环境。
2. 模型超时/OOM：并发降为 1，按 `docs/guides/02_运维手册.md` 降级；确认使用 `qwen3:8b`（非旧版 `deepseek-r1:8b`）。
3. 环境变量错误：核对 `.env` 必填项与格式。
4. Ollama 模型拉取失败：检查网络/代理，也可先用 `ollama list` 确认是否已有 `qwen3:8b`。
5. AI 调用失败：确认 ai-api 四个 Web 服务已启动且手动登录有效，或 Codex/Gemini CLI 已配置；本部署**不使用** API Key，见 `docs/core/08_AI接入策略.md`。
6. 北向/东财接口报 `ConnectionResetError`：系统代理会拦截国内请求；代码已内置 `bypass_proxy()`，若仍失败可临时关闭 Clash 等代理再试。

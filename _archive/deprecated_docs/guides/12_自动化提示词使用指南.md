# 自动化提示词使用指南

> **文档编号**：`docs/guides/12_自动化提示词使用指南.md`（guides-12）  
> 用途：如何用提示词**真正发现并解决**文档与实现的差距。

---

## 1. 一键执行（推荐）

**步骤**：
1. 打开 `docs/提示词/01_全功能自动化主流程.md`
2. 复制「可直接复制的完整提示词」整段（从 `【角色】` 到 `【禁止】` 末尾）
3. 粘贴到 Cursor 新对话，发送
4. AI 将按 Phase 0 → Phase 1 → Phase 2 执行

**预期**：产出《未实现项清单》《验证证据摘要》《执行摘要》《人类抽检清单》等；每步须粘贴 pytest/grep 实际输出，否则视为未执行。

---

## 2. 多轮执行（提高发现与解决质量）

为提高「真的发现、真的解决」的概率，建议：

### 方式 A：发现与实现分轮

1. **第一轮（仅发现）**：打开 `docs/提示词/13_前置_差距分析_强制代码验证.md`，复制其完整提示词，执行。
2. 检查《验证证据摘要》是否**包含粘贴的 pytest 最后 30 行**和**每项关键 grep 的粘贴结果**。若无，要求 AI 补全。
3. **第二轮（仅实现）**：**开新对话**，输入《未实现项清单》，并说：「请按 docs/提示词/05_循环_代码生成.md 逐项实现上述清单中的可执行项，每项须补充测试用例。先实现 P0，再 P1。」
4. 实现完成后，运行 `pytest tests/ -v` 并对照《人类抽检清单》人工抽检 3～5 项。

### 方式 B：全流程多轮

1. 执行 01 全功能自动化主流程，跑完 Phase 2
2. 若《未实现项清单（终版）》仍非空，**再执行一次** 01 主流程（Phase 0 会重新差距分析，可发现上轮漏检或新增问题）
3. 重复 until 清单为空或仅剩不可自动化项

---

## 3. 关键检查点（人工复核）

执行后，请确认：

| 检查项 | 若无则说明 AI 未严格执⾏ |
|--------|--------------------------|
| Phase 0 输出中有 **pytest 最后 30 行的粘贴** | 13 未真正运行 pytest |
| Phase 0 输出中 **admin/users、auth/refresh、market_state 等逐项有 grep 结果粘贴** | 13 未真正逐项 grep |
| 《未实现项清单》表格「实现状态（证据）」列**每行都有具体引用**（如 grep 文件:行号） | 13 可能凭文档臆断 |
| 06 输出中有 **pytest 最后 30 行的粘贴** | 06 未真正运行 pytest |
| 新实现的 API/功能在 tests/ 中有**对应新增用例** | 05 未遵守「新功能须补测」 |
| Phase 2 有《人类抽检清单》 | 缺少人工快速验证指引 |

---

## 4. 单独使用

| 需求 | 操作 |
|------|------|
| 只做差距分析 | 复制 `13_前置_差距分析_强制代码验证.md` 的完整提示词执行 |
| 只实现某几项 | 把《未实现项清单》中的项整理为任务，复制 05 的提示词 + 任务描述执行 |
| 方案文档有问题 | 用 `docs/提示词.md` 中的「方案深度诊断」「方案修复与迭代」 |
| 文档 vs 实现对照 | 用 `docs/提示词.md` 中的「文档 vs 实现 对照」 |

## 5. 常见问题

**Q：AI 没有粘贴 pytest 输出怎么办？**  
A：回复：「请按 13 要求粘贴 pytest 输出的最后 30 行，无粘贴视为未执行。」

**Q：清单标「无未实现项」但我知道有漏的？**  
A：要求 AI 对 admin/users、auth/refresh、market_state、ReportHighNegativeFeedback 逐项执行 grep 并粘贴结果；或人工跑 `grep -r "admin/users" app/api/` 等验证。

**Q：实现后 pytest 通过但功能不对？**  
A：可能是测试未覆盖该功能。要求 05 补充测试用例；或按《人类抽检清单》人工验证，将问题反馈给 AI 修复。

**Q：多次执行仍漏检？**  
A：尝试「发现与实现分轮」（方式 A），在新对话中仅输入《未实现项清单》做实现，减少同一 AI 自我验证偏差。

## 6. Cursor 并行 Agent 加速（可选）

Cursor 2.0+ 支持 **Worktree** 与 **Best-of-N**，可缩短耗时或提升质量：

| 模式 | 适用 | 操作 |
|------|------|------|
| **Best-of-N** | Phase 0（13）或 05 单任务 | 在 Composer 中启用多模型并行（2～4 个），取清单最全或实现最佳版本 |
| **多任务并行** | 多个无依赖、文件不重叠的子任务 | 04 拆解后，开 2～4 个 Composer 会话，每会话执行一条子任务的 05，最后合并 |
| **Worktree** | 每 Agent 独立工作区 | 项目根或 `.cursor/worktrees.json` 已配置依赖安装；并行时 Cursor 自动使用 |

**详见**：`docs/提示词/15_并行执行模式.md`。

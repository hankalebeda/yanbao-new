## 1. 选股方法（一步到位）

### 1.1 主流程

```
东财 clist 全A（push2.eastmoney.com/api/qt/clist/get）
  → 按 f6 成交额降序
  → 应用隔离规则（排除 ST/退市等）
  → 剩余为「全A可用列表」
  → 对全A可用列表应用：进入门槛 + 退出过滤
  → 多因子排序（f6 + 事件活跃度加分）
  → 行业分散约束（可选）
  → 核心池 = min(POOL_SIZE, 通过进入+退出的数量)；不足 200 时取全部通过者
  → 轮转池 = 全A可用列表 - 核心池（即全A股除核心池外的全部）
  → 写入 data/stock_pool_cache_{trade_date}.json（含 core + rotation）
  → 写 runtime/logs/stock_pool_changes_{date}.txt
```

### 1.2 隔离规则（必须排除，核心池与轮转池均适用）

| 规则 | 判定条件 | 数据来源 |
|------|----------|----------|
| ST/退市 | 名称含 ST/*ST/退/退市 | 东财 f14 / company_data |
| 上市不足 1 年 | listed_date 距今日 < 365 天 | company_data.fetch_company_overview |
| 市值 < 30 亿 | 总市值 < 30e8 | 东财 f20 或 price×总股本 |
| 60 日 3 日+ 连续跌停 | 近 60 日 K 线连续 ≥3 日跌停 | tdx_local_data / 东财 K 线 |
| 审计意见非标 | 年报审计意见非标准 | company_data（可配置关闭）|

### 1.3 进入门槛（仅核心池适用；轮转池不要求）

| 门槛 | 阈值 | 数据来源 |
|------|------|----------|
| 日均成交额 | 近 20 日日均成交额 ≥ 2 亿 | 东财历史 / tdx |
| 单日成交额 | 当日 f6 ≥ 1 亿 | clist 实时 |
| 机构持仓 | ≥ 10% | 东财/财报（数据可得时启用）|

### 1.4 退出过滤（仅核心池适用；轮转池不要求）

| 规则 | 阈值 | 数据来源 |
|------|------|----------|
| 流动性枯竭 | 连续 10 日日均成交额 < 2 亿 | 历史成交额 |
| 退市风险 | 已在隔离规则覆盖 | — |

### 1.5 排序与截断

- **全A可用列表**：隔离后的全A，按 f6 成交额降序
- **核心池候选**：对全A可用列表应用进入+退出，再按 f6 + 事件活跃度加分排序
- **核心池**：取前 POOL_SIZE 只（默认 200）
- **轮转池**：全A可用列表 - 核心池；按 f6 成交额降序排列（流动性高的先执行）

### 1.6 数据依赖

| 数据 | 用途 | 来源 | 必需 |
|------|------|------|------|
| 全A列表 + f6 成交额 | 核心池选股、轮转池构建 | 东财 clist | ✅ |
| 公司名称 | ST/退 隔离 | 东财 f14 / company_data | ✅ |
| 上市日期 | 上市<1年隔离 | company_data | ✅ |
| 总市值 | 市值<30亿隔离 | 东财 f20 或估算 | ✅ |
| 近 60 日 K 线 | 连续跌停隔离 | tdx_local_data / 东财 | ✅ |
| 近 20 日成交额 | 核心池进入/退出 | clist 历史 / tdx | ✅ |
| 热搜/龙虎榜 | 事件活跃度加分 | 现有采集 | 可选 |
| 行业分类 | 行业分散 | 东财 | 可选 |
| 机构持仓 | 机构加分 | 东财/财报 | 可选 |
| 全A拉取 | 分页与失败降级 | 东财 clist 支持分页；全量失败时 rotation=[]，或仅用已有候选 |

## 2. 股票池（核心池 + 轮转池）

### 2.1 池组成

```
核心池：选股输出前 200 只（POOL_SIZE），经过隔离+进入+退出+排序
轮转池：全A股 - 核心池 = 全A可用列表（隔离后）中除核心池外的全部股票
持仓延续池：sim_position.status=OPEN 的 stock_code
```

**轮转池定义**：A 股全市场中，通过隔离规则、且不在核心池内的所有股票。规模约 4000～5000 只（视隔离后数量）。**耗时参考**：核心池 200 份约 8～10h；轮转池全量需数十小时至数日，可跨日分批执行，无强制截止。

### 2.2 当日研报生成列表（优先级）

| 优先级 | 池 | 说明 |
|--------|-----|------|
| 1 | 核心池 200 + 持仓延续池 | 15:20 启动，当日夜里完成（可延至次日 02:00）|
| 2 | 轮转池 | 核心池全部完成后，空闲时按序执行，**无时间限制** |

### 2.3 核心池与持仓延续

- **合并逻辑**：`get_daily_stock_pool(include_holdings=True)` 返回「核心池 200 ∪ 持仓延续池」去重列表
- **规则**：凡 `sim_position.status=OPEN` 的 `stock_code` 自动加入当日核心池研报任务

### 2.4 轮转池（空闲时执行）

| 项 | 说明 |
|------|------|
| **标的** | 全A股 - 核心池（即全A可用列表中除核心池外的全部）|
| **规模** | 约 4000～5000 只（隔离后全A 减去 200）|
| **触发** | 核心池全部完成后；或调度器判定「空闲」时 |
| **顺序** | 按 f6 成交额降序依次执行（流动性高的优先）|
| **时间限制** | **无**；跨日、跨周均可，处理完即停；中断后下次选股日可续跑（进度续用或清空，见 §2.6） |
| **执行范围** | 尽力覆盖，不要求全量跑完；可配置 `ROTATION_POOL_TOP_N`（0=不限制，>0=仅跑前 N 只）|
| **模型** | 与核心池相同（可配置为本地预判 + Top-N 升级以节省 API）|
| **配置** | `ROTATION_POOL_ENABLED`、`ROTATION_POOL_USE_SAME_MODEL` |

### 2.5 持仓股被 ST/退市

| 动作 | 说明 |
|------|------|
| 结算 | sim_settle 照常处理，不因出池而中断 |
| 告警 | 检测到持仓股名称含 ST → 发送 admin 通知 |
| 强制平仓 | 可选配置，默认不启用 |

### 2.6 边界与兜底

| 场景 | 策略 |
|------|------|
| 选股任务失败 | 沿用上一交易日池；若无则回退 `STOCK_POOL`；`stock_pool_fallback` 打日志 |
| STOCK_POOL_OVERRIDE 已配置 | 覆盖自动选股，直接使用（仅核心池；轮转池 = 全A - 该覆盖列表）|
| STOCK_POOL_EXTRA | 额外股票合并入核心池，去重 |
| 冷启动 | 首日将 OPEN 持仓 + 近 7 日有研报的 stock_code 加入持仓延续池 |
| 池变更 | `runtime/logs/stock_pool_changes_{date}.txt` 记录 added/removed |
| 轮转池未完成 | 下次选股日会重新产出轮转列表；可保留 T 日进度续跑或清空（实现时二选一）|
| 选股失败时 rotation | rotation=[] |
| 轮转池研报 trade_date | = 选股日（与核心池一致）|

## 3. 研报调度（双轨 + 轮转）

### 3.1 日度研报（核心池）

| 项 | 说明 |
|------|------|
| 触发 | 15:20 |
| 标的 | 核心池 200 + 持仓延续池 |
| 截止 | 当日夜里或次日 02:00（DAILY_JOB_DEADLINE_HOUR）|
| 容量 | 约 200～220 份（含持仓延续）|
| 时间 | 8～10h 可完成 |

### 3.2 盘中研报（可选）

| 项 | 说明 |
|------|------|
| 触发 | 10:00、11:00、14:00、15:00 |
| 标的 | 仅 `sim_position.status=OPEN` 的股票 |
| 用途 | 盘中观点变化 → 研报 SELL → 平仓 |
| 配置 | HOURLY_UPDATE_ENABLED、HOURLY_UPDATE_HOURS |
| 幂等 | `hourly:{stock_code}:{date}:{hour}`；FR-03「单股单日 1 份」仅指 daily 正式报告 |

### 3.3 轮转池研报（空闲时）

| 项 | 说明 |
|------|------|
| 触发 | 核心池全部完成后；或空闲信号 |
| 标的 | 轮转池（全A股 - 核心池）|
| 顺序 | 按 f6 成交额降序依次执行 |
| 时间限制 | 无；可跨日、跨周 |
| 配置 | ROTATION_POOL_ENABLED |

### 3.4 调度关系

- **替代原 tier2**：不再使用 17:00 固定批次，改为「轮转池空闲时执行」
- 日度、盘中、轮转均调用 `generate_report`，`run_mode` 区分（daily / hourly / rotation）
- 核心池与轮转池互斥：同一 (stock_code, trade_date) 只生成一次研报

## 4. 模拟实盘

### 4.1 开仓

- **条件**：recommendation=BUY 且 confidence≥0.65，market_state 非 BEAR 过滤 B/C，drawdown_state≠HALT
- **三档资金**：1w / 10w / 50w 独立账户，每档独立持仓与快照
- **仓位**：Kelly 公式，1w×0.5、10w×1.0、50w×0.7
- **实现**：`create_position` 写入 `sim_position`，T+1 日 09:35 填充 `actual_entry_price`

### 4.2 资金档位

| 档位 | 初始本金 | 仓位系数 | 说明 |
|------|----------|----------|------|
| 1w | 10,000 元 | ×0.5 | 手续费占比大，更保守 |
| 10w | 100,000 元 | ×1.0 | 默认档 |
| 50w | 500,000 元 | ×0.7 | 大资金分散，单票略降 |

### 4.3 平仓优先级

```
1. 止损（价触及 stop_loss_price）
2. 目标价（target_price_2 或 target_price_1 且 hold_days≥5）
3. 研报 SELL（日度/盘中任一研报 recommendation=SELL）
4. 超时（valid_until 或 hold_days 超限）
```

- **新增 status**：`CLOSED_REPORT_SELL`，与 CLOSED_SL / CLOSED_T1 / CLOSED_T2 / CLOSED_EXPIRED 并列

### 4.4 日度结算（15:30）

- 遍历 OPEN 持仓，按优先级判断平仓
- 研报 SELL 检查：该股当日或最近研报 recommendation=SELL → 平仓
- 净盈亏扣除手续费（万三）和印花税（0.05%）
- 写入 `sim_account` 每档日度快照

## 5. 时间线与数据流

```
T 日（交易日）
│
├─ 09:00  选股 refresh_stock_pool → stock_pool_cache_{T}.json（core + rotation）
├─ 09:00  市场状态 calc_and_cache_market_state
├─ 09:35  更新 T+1 开盘价（actual_entry_price）
├─ 10/11/14/15  盘中研报（可选）→ 仅 OPEN 持仓股
├─ 15:20  日度研报（核心池）→ 200 + 持仓延续 → BUY 开仓 / SELL 平仓
├─ 15:30  日度结算 run_settle → 止损/目标/研报SELL/超时
└─ 核心池完成后  轮转池研报（可选）→ 全A-核心池，按序执行，无时间限制
```

## 6. 模块改造要点

### 6.1 选股模块（新建或扩展）

```
refresh_stock_pool(trade_date) -> dict
  1. 拉取东财 clist 全A，按 f6 降序
  2. 隔离：ST/退、上市<1年、市值<30亿、60日3连跌停 → 得「全A可用列表」
  3. 对全A可用列表应用进入+退出，排序（f6 + 事件活跃度）
  4. 行业分散（可选）
  5. core = min(POOL_SIZE, 通过数量)；不足时取全部通过者
  6. rotation = 全A可用列表 - core；全A拉取失败时 rotation=[]
  7. 写 stock_pool_cache_{date}.json（格式：{"core": [...], "rotation": [...]}）
  8. 写 stock_pool_changes_{date}.txt
```

### 6.2 stock_pool.py

```
get_daily_stock_pool(trade_date, include_holdings=True, name_resolver, pool_type="core")
  pool_type="core"：返回当日核心池（用于 15:20 日更）
  pool_type="rotation"：返回当日轮转池（全A - 核心池，用于空闲任务）
  pool_type="all"：返回 core + rotation（可选）
  1. STOCK_POOL_OVERRIDE → 解析返回（仅 core；rotation = 全A - 该列表）
  2. 读 stock_pool_cache_{trade_date}.json
  3. 无缓存 → 回退 STOCK_POOL（仅 core，rotation 可现场计算 全A - core 或为空）
  4. include_holdings → 合并 sim_position.status=OPEN（仅对 core）
  5. filter_by_isolation（对 core 返回结果再过滤一次）
  6. 返回去重列表
```

### 6.3 scheduler.py

| 任务 | 时间 | 说明 |
|------|------|------|
| _stock_pool_refresh_job | 09:00 | 新增，调用 refresh_stock_pool |
| _market_state_job | 09:00 | 已有 |
| _sim_open_price_job | 09:35 | 已有 |
| _hourly_holdings_job | 10/11/14/15 | 新增，可选，仅 OPEN 持仓股 |
| _daily_job_entry | 15:20 | 调用 get_daily_stock_pool(pool_type="core", include_holdings=True) |
| _sim_settle_job | 15:30 | 增加研报 SELL 平仓分支 |
| _rotation_pool_job | 空闲时 | 新增，核心池完成后按序跑轮转池（全A-核心池）|

### 6.4 report_engine.py

- BUY 强信号 → create_position（已有）
- 研报 SELL：生成后检查该股是否有 OPEN 持仓，若有 → 调用平仓或打标供 settle 执行

### 6.5 sim_settle_service.py

- 在止损/目标价判断之后、超时之前，增加「研报 SELL」分支
- 查询该股当日/最近研报 recommendation=SELL → status=CLOSED_REPORT_SELL

## 8. 验收要点

### 8.1 选股

| 项 | 标准 |
|------|------|
| 隔离规则 | ST/退、上市<1年、市值<30亿、60日3连跌停股不出现在 core 与 rotation |
| 进入门槛 | 核心池通过流动性门槛；轮转池不要求 |
| 核心池 | 选股成功后 core 为 200 只 |
| 轮转池 | 选股成功后 rotation = 全A可用列表 - core |
| 变更日志 | stock_pool_changes 有 added/removed |
| 选股失败 | 回退 STOCK_POOL，打日志 |

### 8.2 核心池

| 项 | 标准 |
|------|------|
| 每日全覆盖 | 核心池 200 只全部每日获得完整研报 |
| 持仓延续 | 有 OPEN 持仓时，该股必在研报生成列表 |
| 时间窗口 | 200 份研报在夜里或次日 02:00 前完成 |

### 8.3 轮转池

| 项 | 标准 |
|------|------|
| 定义 | 全A股 - 核心池（通过隔离规则的全A，减去核心池 200）|
| 触发 | 核心池全部完成后开始执行 |
| 顺序 | 按 f6 成交额降序依次执行 |
| 无时间限制 | 可跨日、跨周，无强制截止 |

### 8.4 模拟实盘

| 项 | 标准 |
|------|------|
| BUY 开仓 | confidence≥0.65 且非 filtered_out 时创建 sim_position |
| 研报 SELL 平仓 | 研报 SELL 且该股有 OPEN 持仓时平仓，status=CLOSED_REPORT_SELL |
| 平仓优先级 | 止损 > 目标价 > 研报 SELL > 超时 |
| 三档资金 | 每档独立持仓、独立快照、可切换展示 |

### 8.5 Admin 展示

| 项 | 标准 |
|------|------|
| 股票池 | 展示来源（自动/覆盖）、核心池规模 200、轮转池规模（全A-200）、当日快照 |
| 轮转池 | 可选展示进度（已处理数/总数）|
| 列表页 | 默认展示核心池+持仓延续的研报；轮转池研报是否纳入及展示方式单独约定 |

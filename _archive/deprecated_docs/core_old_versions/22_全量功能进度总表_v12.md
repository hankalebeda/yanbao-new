## 9. 维护规则

### 9.1 更新原则

1. **真实系统完成率**：每次更新必须基于 HTTP 实测 + DB 审计 + LLM 连通测试 + pytest 回归，**禁止仅凭路由可达性统计**
2. **六维度评估**：必须同时评估 D1-D6 六个维度，空表和缺失字段作为"未完成"计入
3. **证据要求**：每个判定附可验证证据（HTTP 状态码 / DB 查询 / pytest 输出）
4. **唯一 AI 来源**：NewAPI Gateway GPT；运行时 SSOT 为 `ai-api/codex/*` provider 目录，`ubuntu.txt` 不得作为调度配置来源

### 9.2 文档管理

5. **单一权威文件**：本文件（v12）为唯一进度总表，不得创建新的 v13/v14 等分支版本
6. **就地更新**：问题变更直接更新第 7 节；已修复移入"已关闭"小节，不删除
7. **数据快照**：每次更新在附录追加一行版本记录，包含日期和关键指标差异
8. **禁止膨胀**：变更日志不超过 3 行/次；历史修复记录压缩为一行摘要

### 9.3 后续优化规则

9. **修复优先级排序**：数据补充（D2/D3 提升）> 前端体验（P1）> 外部接入（P2/外部阻塞）
10. **每次修改后验证**：`python -m pytest tests/{affected_test_file} -q --tb=short`，禁止引入新 failure
11. **功能点状态变更**：⚠️→✅ 需同时提供 API 返回值 + DB 行数 + 页面截图三线证据，缺任一不得升级
12. **外部阻塞项**：不做代码降级，标记为"外部阻塞"等待接入
13. **研报生成单一路径**：公开单报告入口仅允许 `/api/v1/reports/generate` → `report_generation_ssot.py`；`/api/v1/internal/reports/generate` 已退役，内部仅保留 batch/tasks 编排接口
14. **研报零缺数据**：生成前必须校验 `kline_daily / hotspot_top50 / northbound_summary / etf_flow_summary / market_state_input` 全齐且 `status=ok`；任一缺席或非 ok 一律 `REPORT_DATA_INCOMPLETE` 失败，并物理删除不合格研报及其附属记录
15. **正式文档目录洁净**：`docs/core` 仅允许正式编号 Markdown；脚本、测试结果、临时核验产物必须移出到 `tests/`、`_archive/` 或 `output/`

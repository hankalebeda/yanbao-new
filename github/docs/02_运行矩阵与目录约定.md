# 02_运行矩阵与目录约定

## 目录约定

| 路径 | 用途 | 备注 |
| --- | --- | --- |
| `github/automation/continuous_audit/` | 当前连续巡检输出 | 包含 `latest_run.json`、`continuous_audit_issue_ledger.md`、`history/` |
| `github/automation/live_fix_loop/` | 当前 live fix loop 台账与状态 | 包含 `issue_register.md`、`review_log.md`、`automation_runs/`、`automation_state.json` |
| `github/automation/runs/<run_id>/` | guardian worktree 内的标准运行产物路径 | 作为单次 run 的标准归档形态 |
| `github/automation/_local/runs/<run_id>/` | 主仓库上的本机镜像产物 | 便于本机追踪，不是规格来源 |
| `github/automation/_local/latest.json` | 指向最近一次本机镜像 run | 只做索引 |
| `github/docs/` | 长期保留的说明文档 | 可提交、可评审 |

## 运行矩阵

| 脚本 | 运行方式 | 默认输出根 | 是否改代码 |
| --- | --- | --- | --- |
| `scripts/run_continuous_repo_audit.ps1` | 手动/计划任务 | `github/automation/continuous_audit/` | 否 |
| `scripts/live_fix_loop.py init` | 手动/编排器调用 | `github/automation/live_fix_loop/` | 否 |
| `scripts/run_codex_prompt6_hourly.ps1` | 手动/计划任务 | `github/automation/live_fix_loop/automation_runs/` | 可能 |
| `scripts/run_github_guardian.ps1` | 手动/计划任务 | `github/automation/runs/` + `_local/` | 可能 |
| `scripts/run_issue_mining_22.ps1` | 手动 | `runtime/issue_mining_22/` | 否 |

## 单一对外入口

- GitHub 自动化的长期说明只看 `github/README.md` 与 `github/docs/**`。
- GitHub 自动化的当前运行状态只看 `github/automation/continuous_audit/` 与 `github/automation/live_fix_loop/`。
- `docs/_temp/continuous_audit/`、`docs/_temp/live_fix_loop/` 仅作为旧路径兼容来源，不再作为对外入口。

## 兼容迁移策略

- `continuous_repo_audit.py`、`live_fix_loop.py`、`codex_prompt6_hourly.py` 会优先使用 `github/automation/**`。
- 若检测到旧的 `docs/_temp/continuous_audit/` 或 `docs/_temp/live_fix_loop/` 且新目录不存在，脚本会复制旧目录内容作为迁移种子。
- 迁移完成后，新的读写都以 `github/automation/**` 为准。

## 评审口径

- `github/docs/**` 属于可发布文档。
- `github/automation/**` 属于运行产物，不应被当作业务文档或业务规格。
- `_local/` 下的内容只为本机观测和故障复盘保留。

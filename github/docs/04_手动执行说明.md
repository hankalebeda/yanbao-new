# 04_手动执行说明

## 结论

可以，不需要注册任何本机计划任务。

当前 GitHub 自动化链路已经支持纯手动执行，直接运行 PowerShell 入口即可。

## 推荐入口

### 1. 手动执行完整 guardian 链路

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-and-fix -NoPush
```

说明
- 这是最接近“手动触发一次自动优化和迭代”的入口。
- 会串联 `continuous_repo_audit.py`、`live_fix_loop.py init`、`codex_prompt6_hourly.py`。
- 会真实启用 `codex exec --enable multi_agent`，也就是子代理能力。
- `-NoPush` 表示只在本地跑，不向远端推送。

### 2. 只手动执行巡检

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_continuous_repo_audit.ps1
```

输出目录
- `github/automation/continuous_audit/`

### 3. 只手动执行 Prompt 6 修复闭环

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_codex_prompt6_hourly.ps1
```

输出目录
- `github/automation/live_fix_loop/`

## 常用手动参数

### 只巡检不修复

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-only -NoPush
```

### 保留 worktree 便于人工复盘

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-and-fix -NoPush -KeepWorktree
```

### 禁止自动拉起 8000 运行时

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-and-fix -NoPush -NoEnsureRuntime
```

### 只做本地 dry run

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-and-fix -NoPush -DryRunFix
```

## 手动执行后的主要产物

- `github/automation/continuous_audit/`
- `github/automation/live_fix_loop/`
- `github/automation/_local/latest.json`
- `github/automation/_local/runs/<run_id>/manifest.json`

## 不需要执行的脚本

在手动模式下，不需要运行以下计划任务注册脚本：

- `scripts/register_github_guardian_task.ps1`
- `scripts/register_codex_prompt6_hourly_task.ps1`
- `scripts/register_continuous_repo_audit_task.ps1`

## 建议口径

如果你的目标是“我手工点一次，然后让它用子代理自动跑一轮优化和迭代”，优先使用这一条：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\run_github_guardian.ps1 -Mode audit-and-fix -NoPush
```

# 28_严格验收与上线门禁

> **文档编号**：28  
> **文档名称**：严格验收与上线门禁  
> **项目名称**：A 股个股研报平台（`yanbao-new`）  
> **版本**：v1.0  
> **产出日期**：2026-04-26  
> **用途**：为 Ralph Step 2 提供验收门禁补充；runtime 直接入口仍是 `.claude/ralph/loop/prd.json`。

---

## 1. 总原则

- `passes` 只能在真实实现完成且对应验收通过后从 `false` 改为 `true`。
- 任何 blocked、外部依赖缺失、数据源不可用、JSON 解析失败、测试失败或浏览器验证失败，都必须 fail-close，不得手工翻转 `passes`。
- 未获用户当轮明确授权时，Ralph 不得执行 `git checkout`、`git commit`、`git reset` 或任何分支改写动作。
- 禁止写入 `scripts/`、`data/`、`output/`，禁止在仓库根目录新增临时文件。
- `docs/core/08_AI接入策略.md` 当前不存在；不得发明该文档内容，必要规则必须内联到 27 与 `prd.json`。

---

## 2. 文档与 JSON 门禁

- `docs/core/27_PRD_研报平台增强与整体验收基线.md`、`.claude/ralph/loop/prd.json`、`.claude/ralph/prd/yanbao-platform-enhancement.json` 必须描述同一个目标系统。
- 两份 `prd.json` 必须字节级或结构级一致；runtime 版是 Step 2 唯一直接任务入口。
- `prd.json` 只允许 Ralph 最小 schema：`project`、`branchName`、`description`、`userStories`，以及 story 的 `id`、`title`、`description`、`acceptanceCriteria`、`priority`、`passes`、`notes`。
- 每条 story 的 `notes` 必须是可解析的紧凑 JSON 字符串，且至少包含 `group`、`dependsOn`、`endpoints`、`models`、`permissions`、`errorCodes`、`idempotency`、`enums`、`thresholds`、`degradation`、`exampleAssert`、`pytest`。

---

## 3. 单轮验收门禁

- Ralph 每轮只能选择最高优先级且 `passes=false` 的一条 story。
- 只允许修改该 story 必需文件；无关重构、顺手修复和跨 story 实现均视为越界。
- 代码 story 必须运行 story 中声明的最小 pytest 切片；仅 Markdown/JSON/PowerShell 变更至少要完成 JSON parse、双份同步和 runner dry-run。
- UI story 必须完成浏览器验证；若浏览器工具不可用，story 保持 `passes=false` 并在 `progress.txt` 写明阻塞。
- 外部数据、OAuth、支付、LLM provider 不可用时必须保留 blocked/degraded 事实，不得用 mock、常量或历史缓存伪装可用。

---

## 4. 推荐验证命令

- JSON 与双份同步：解析 `.claude/ralph/loop/prd.json` 与 `.claude/ralph/prd/yanbao-platform-enhancement.json`，比较结构完全一致。
- Runner dry-run：`powershell -ExecutionPolicy Bypass -File .claude/ralph/run-ralph.ps1 -Tool claude -MaxIterations 1 -DryRun`。
- 代码切片：执行当前 story `acceptanceCriteria` 中声明的 `python -m pytest ... -q --tb=short`。
- 全量验证只在需要时执行：`python -m pytest tests/ -q --tb=short`。

---

## 5. 通过定义

一条 story 只有同时满足以下条件才可置为 `passes=true`：

- 实现范围与该 story 完全一致；
- 验收命令通过；
- 没有新增 forbidden path、根目录临时文件或未授权 git 操作；
- 相关文档、runtime `prd.json`、命名副本和 `progress.txt` 已同步；
- 失败态、降级态和外部阻塞均被显式保留。
